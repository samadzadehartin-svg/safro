# ساختار کد سفرو

این نسخه از نظر فرمت و خوانایی مرتب‌سازی شده و ظاهر/رفتار نسخه قبل حفظ شده است.

## پوشه‌ها

- `index.html`: صفحه ورودی اصلی سایت
- `buyer/index.html`: مسیر صفحه مشتری
- `admin/index.html`: مسیر پنل مدیریت
- `staff/index.html`: مسیر پنل کارشناس/فروش
- `assets/css/style.css`: استایل اصلی کل پروژه
- `assets/js/core.js`: منطق مشترک، داده‌های پایه، ذخیره‌سازی، ابزارها و رندرهای عمومی
- `assets/js/buyer.js`: منطق صفحه مشتری، تورها، کشورها، جزئیات، رزرو، فیلترها و Back مرورگر
- `assets/js/admin.js`: منطق پنل مدیریت
- `assets/js/staff.js`: منطق پنل کارشناس
- `assets/js/data.js`: داده‌های اولیه تورها، ویزا، سفارش‌ها و تنظیمات
- `assets/js/config.js`: تنظیمات اتصال Supabase
- `assets/images/`: لوگو، تصاویر و SVGهای لاین‌آرت
- `sw.js`: سرویس‌ورکر و کش PWA
- `manifest.json`: تنظیمات نصب وب‌اپ

## نکات نگهداری

- برای تغییرات صفحه مشتری، اول `assets/js/buyer.js` و سپس بخش‌های مربوط در `style.css` را بررسی کن.
- برای تغییر داده‌های اولیه تورها و مقصدها، `assets/js/data.js` محل اصلی است.
- برای تغییر ظاهر عمومی کارت‌ها، هدر، دکمه‌ها و بک‌گراند، `assets/css/style.css` محل اصلی است.
- بعد از هر تغییر مهم، مقدار `CACHE_NAME` در `sw.js` را تغییر بده تا مرورگر فایل‌های قدیمی را از کش نگه ندارد.

## فرمت کد

فایل‌های HTML/CSS/JS/JSON/Markdown با Prettier مرتب شده‌اند. فایل `.prettierrc` داخل پروژه تنظیمات فرمت را نگه می‌دارد.
